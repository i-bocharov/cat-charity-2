# 🐱 QRKot — Backend для благотворительного фонда

REST API для управления целевыми проектами и автоматического распределения пожертвований фонда поддержки котиков QRKot.

## 🛠 Технологический стек
- Язык: Python 3.12
- Фреймворк: FastAPI (асинхронный)
- ORM: SQLAlchemy 2.0 (asyncio)
- База данных: SQLite
- Аутентификация: fastapi-users + JWT
- Валидация: Pydantic v2
- Миграции: Alembic
- Тесты: pytest, pytest-asyncio

## ✨ Основные возможности
- Управление проектами: CRUD для целевых сборов. Автоматический расчёт invested_amount, fully_invested и дат создания/закрытия.
- Пожертвования: Создание донатов с опциональным комментарием. Система автоматически распределяет средства по старейшим открытым проектам (FIFO-алгоритм).
- Ролевая модель: JWT-аутентификация. Обычные пользователи видят только свои пожертвования, суперпользователи управляют проектами и финансовой сводкой.

## 📁 Структура проекта
```
app/
├── api/            # Эндпоинты, валидаторы бизнес-правил
├── core/           # Конфигурация, БД, аутентификация, UserManager
├── crud/           # Базовые и кастомные CRUD-операции
├── models/         # SQLAlchemy ORM-модели (абстрактный базовый класс)
├── schemas/        # Pydantic-схемы запросов/ответов
└── services/       # Бизнес-логика (инвестирование пожертвований)
```

## 🚀 Быстрый старт

### 1. Клонирование и виртуальное окружение

`git clone <url-твоего-репозитория>`

`cd cat-charity-2`

`python -m venv venv`

# Linux/macOS: `source venv/bin/activate`
# Windows: `venv\Scripts\activate`

### 2. Установка зависимостей
`pip install -r requirements.txt`

### 3. Переменные окружения (опционально)
Создайте файл .env в корне проекта в соответствии с образцом .env.example
(Если файл отсутствует, приложение использует безопасные значения по умолчанию)

### 4. Применение миграций
`alembic upgrade head`

### 5. Запуск сервера
`uvicorn app.main:app --reload`
API будет доступно по адресу: `http://127.0.0.1:8000`

## 📖 Документация API
После запуска доступны интерактивные спецификации:
- Swagger UI: `http://127.0.0.1:8000/docs`
- ReDoc: `http://127.0.0.1:8000/redoc`
- OpenAPI JSON: `http://127.0.0.1:8000/openapi.json`

## 🧪 Тестирование
Запуск полного набора тестов:
`pytest tests/ -v --tb=short`
Тесты используют изолированную SQLite-базу в памяти. Таблицы автоматически создаются перед каждым тестом и удаляются после него.

## 🔐 Безопасность и Best Practices
- Строгая типизация (mypy), полное отсутствие any
- Валидация входных данных через Pydantic v2 (extra=forbid) для предотвращения передачи лишних полей
- Корректные HTTP-статусы (400 для бизнес-правил, 422 для схем)
- Транзакционная целостность: инвестирование и обновление статусов происходят в рамках одной session.commit()
- Архитектура: слоистая (API → Service → CRUD → DB), соблюдение SRP, использование DI и Annotated
